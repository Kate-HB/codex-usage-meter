# Codex 用量悬浮窗设计

## 目标

在 Windows 上显示轻量、置顶的 Codex 用量悬浮窗。Codex 运行时自动显示，退出后隐藏，无需用户反复打开用量页面。

## 范围

- 显示周额度剩余百分比、进度条和重置时间。
- 显示当前会话上下文剩余百分比和进度条。
- 监听本机 Codex 会话日志，使用其中的官方 `rate_limits`、`token_count` 和 `model_context_window` 数据，不联网、不估算账户额度。
- 支持拖动和 `Ctrl + 鼠标滚轮` 60%–120% 连续缩放。
- 保存窗口位置和缩放比例。
- 后台静默启动；仅检测到 Codex 进程时显示窗口。

## 技术方案

使用 Windows PowerShell 5.1 与 WPF 实现，不引入第三方依赖。启动项只负责拉起隐藏的监视进程；WPF 窗口不进入任务栏并保持置顶。

## 组件

1. 进程监视器：定时检测 Codex Desktop 相关进程，控制窗口显隐。
2. 日志读取器：定位 `%USERPROFILE%\.codex\sessions` 下最近更新的 JSONL 会话文件，只增量读取新增记录。
3. 用量解析器：提取最新限额和 token 信息，计算 `100 - used_percent` 及 `100 - last_token_usage.total_tokens / model_context_window * 100`。
4. 悬浮窗：显示两条始终保留的进度条、重置时间、数据新鲜度和实时状态。
5. 设置存储：在用户本地应用数据目录保存窗口坐标与缩放比例。

## 数据流

监视器发现 Codex 运行后显示窗口；日志读取器跟随最新活跃会话文件。每次出现新的 `token_count` 事件，解析器更新两类剩余量，界面同步刷新。Codex 退出后窗口隐藏，后台监视器继续等待下次启动。

## 交互

- 拖动窗口任意空白区域可移动。
- 悬停窗口并按住 `Ctrl` 滚动鼠标，按固定步长缩放，边界为 60% 和 120%。
- 缩放只改变整体尺寸，两条进度条及其数值始终保留。
- 周额度按健康、提醒、危险三个区间改变颜色；上下文使用独立蓝色。

## 异常处理

- 尚无日志或字段缺失：显示 `等待 Codex 数据`，不显示虚构数值。
- 日志轮换或切换会话：自动切换到最新活跃文件并重新读取最新有效状态。
- 日志暂时无更新：保留上次有效值，并显示最后更新时间。
- 日志行损坏或写入未完成：跳过该行，后续更新时继续解析。
- 保存的坐标位于已断开的显示器外：窗口回到主屏幕右上角。

## 验证

- 用固定 JSONL 样例验证额度、上下文和重置时间计算。
- 验证文件追加、轮换、缺失字段及损坏行。
- 验证 Codex 启停时窗口显隐。
- 验证拖动、缩放边界、设置持久化及屏幕外位置恢复。
- 在 Windows PowerShell 5.1 下执行启动与基本运行检查。

## 完成标准

启动 Codex 后悬浮窗自动出现；两项剩余量随会话日志更新；退出 Codex 后窗口隐藏；重启监视器后位置和比例保持；无需安装额外运行时。
